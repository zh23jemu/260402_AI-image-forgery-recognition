# ADM 案例观察目标

## 1. 用途

这份文档用于统一后续在 `ADM` 案例上最希望观察到什么，不让 `LVLM` 分析结果变成一堆难以整理的自由文本。

## 2. 第一层目标：直接服务论文

后续每个案例最好至少能支持下面三类表述之一：

- 为什么 `ADM` 比 `GLIDE`、`SD` 更难
- 为什么首轮微调虽然有效，但没有彻底解决困难样本
- 为什么第二轮保守微调没有进一步改善

## 3. 第二层目标：直接服务案例解释

每个案例至少希望回答以下一个问题：

- 这张图最可疑的细节是什么
- 这张图最自然的细节是什么
- 为什么不同模型会对这张图给出不同判断
- 为什么这张图会落在边界区域

## 4. 第三层目标：服务后续改进方向

如果一个案例足够典型，最好还能支持下面至少一条观察：

- 当前问题更像是局部结构异常难以稳定捕捉
- 当前问题更像是高层语义不协调但低层纹理不明显
- 当前问题更像是阈值与校准，而不是完全没有区分能力
- 当前问题并不能只靠继续调学习率解决

## 5. 不建议追求的内容

后续做 `LVLM` 时，不建议把重点放在：

- 让模型给出很长很花哨的描述
- 让模型硬性做统一分数排名
- 让模型替代当前所有判别模型

## 6. 当前建议

最值得保留的案例，不一定是回答最长的，而是最能支撑论文里那一句关键解释的。
