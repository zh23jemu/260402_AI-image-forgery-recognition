## 2. 模板 A：强化版训练后出现明确增益

### 2.1 适用条件

- 强化版 `FSD + SP` 至少在 `ADM`、`SD`、`Midjourney` 中一类或多类明显高于 `FSD-only`
- 且日志显示 `valid_sp_samples`、`sp_loss` 等信号正常

### 2.2 可直接写入第五章的结果结论

强化版第一阶段联合训练结果表明，`Stay-Positive` 约束在当前闭集协议下已经开始体现出实际作用。与同协议 `FSD-only` 对照相比，增强版 `FSD + Stay-Positive` 在关键测试类别上的准确率和平均精度出现了可观察提升，说明这种 fake-oriented 判别约束并非只停留在概念层面，而是能够在现有 `FSD` 主干上产生真实增量。从结果性质看，这种增益更多体现为主干模型对目标 fake 证据的响应被进一步强化，从而在困难类别或边界类别上表现出更稳定的识别趋势。

这一现象对本文的联合训练主线具有直接支撑意义。它说明，本文提出的“以 `FSD` 为主干、以 `Stay-Positive` 为判别约束来源、以 `LVLM` 为后续语义辅助接口”的两阶段联合训练思路不仅工程上可落地，而且在第一阶段就已经能够体现出初步性能收益。因此，联合训练部分在本文中不应再被视为单纯的后续设想，而应被视为已经获得初步验证的核心改进方向。

### 2.3 可直接写入第七章的总结结论

第一阶段联合训练的强化版结果进一步表明，`Stay-Positive` 所提供的 fake-oriented 判别约束能够在 `FSD` 主干上产生实际增益。这说明本文提出的联合训练路线不仅完成了框架搭建和工程实现，而且已经在阶段性实验中体现出一定性能提升。基于这一结果，后续引入 `LVLM` 语义辅助监督具有更强合理性，因为第二阶段不再是从零开始尝试，而是在已验证有效的第一阶段联合主干基础上继续扩展。

## 3. 模板 B：强化版训练无明显增益，但约束已明确生效

### 3.1 适用条件

- 强化版 `FSD + SP` 与 `FSD-only` 总体差距不大
- 但日志显示：
  - `valid_sp_samples` 持续非零
  - `sp_loss` 持续非零
  - `steps_with_valid_sp` 明确增长

### 3.2 可直接写入第五章的结果结论

强化版第一阶段联合训练结果显示，`Stay-Positive` 约束已经真实参与到当前训练过程中，但在现有实现和当前闭集协议下，其额外收益仍然有限。与同协议 `FSD-only` 对照相比，增强版 `FSD + Stay-Positive` 没有表现出显著的全局性能提升，但训练日志中的 `valid_sp_samples`、`sp_loss` 以及相关调试信号表明，辅助约束并非失效或未接入，而是已经进入训练计算图，只是当前约束形式尚未转化为明显的指标增量。

这一结果对本文同样具有研究价值。它说明，第一阶段联合训练的主要成果首先是“约束链路已经跑通并完成验证”，而不是“第一阶段已经取得稳定显著增益”。换言之，本文已经证明了联合训练框架的可实现性，但也同时说明，仅依赖 `Stay-Positive` 的离线分数约束还不足以充分释放联合训练潜力。基于这一判断，第二阶段引入 `LVLM` 结构化语义辅助监督，就不再只是补充性想法，而成为更可能带来新增量的关键方向。

### 3.3 可直接写入第七章的总结结论

强化版第一阶段联合训练进一步证明了 `Stay-Positive` 约束链路已经能够真实接入 `FSD` 主干训练过程，但在当前实现下尚未带来显著的整体性能提升。因此，本文关于联合训练的阶段性结论应被概括为：第一阶段已经完成可行性验证和约束链路验证，而更具决定性的增量空间仍然有待第二阶段 `LVLM` 语义辅助监督进一步释放。

## 4. 模板 C：强化版训练后仍基本等价，且约束效果很弱

### 4.1 适用条件

- 强化版 `FSD + SP` 与 `FSD-only` 结果依旧几乎一致
- 同时日志显示：
  - `valid_sp_samples` 很少
  - 或 `sp_loss` 长期极低
  - 或者整体约束信号弱

### 4.2 可直接写入第五章的结果结论

强化版第一阶段联合训练结果表明，在当前实现下，单纯依赖 `Stay-Positive` 离线分数约束尚未有效改变 `FSD` 主干的整体判别表现。与同协议 `FSD-only` 对照相比，增强版训练结果仍保持高度接近，说明当前阶段的主要成果并不是已经获得了明确的联合增益，而是进一步暴露出第一阶段约束形式本身的局限性。结合训练日志可以看出，当前 `SP` 约束的有效作用范围仍然偏弱，这使得其尚未在全局指标上形成足够清晰的区分。

但这一结果并不削弱本文联合训练主线的成立，反而为其提供了更清晰的阶段划分依据。具体而言，第一阶段的价值在于完成联合训练框架落地、验证工程流程可执行，并明确指出“仅靠离线 fake-oriented 分数约束仍不足以带来显著收益”。在此基础上，第二阶段引入 `LVLM` 结构化语义标签，就具备了更明确的问题指向，即通过局部异常类型监督弥补第一阶段仅依赖分数约束的不足。因此，本文最终版仍可以把联合训练写为完整主线，只是需要将第一阶段更准确地界定为“落地验证与问题暴露阶段”。

### 4.3 可直接写入第七章的总结结论

强化版第一阶段联合训练说明，当前联合训练框架已经完成实现与运行验证，但第一阶段仅依赖 `Stay-Positive` 离线分数约束时尚未体现显著增益。这一结果使本文能够更明确地把第二阶段 `LVLM` 语义辅助监督定位为后续最关键的改进方向，也使联合训练的阶段性逻辑更加清晰：第一阶段解决“框架能否落地”，第二阶段解决“如何进一步形成稳定增量”。
