# FSD + Stay-Positive + LVLM 联合设计说明

## 1. 文档定位

本说明用于统一当前项目中“三模型联合训练”的方法口径。

它不再沿用早期“先收束文书、暂不训练”的保守表述，而是明确服务于以下目标：

- 支撑论文主方法章节改写
- 支撑代码端第一版联合训练实现
- 支撑客户对“联合训练是否真实落地”的判断

## 2. 设计目标

本项目的联合设计目标不是简单把三条路线并排摆在一起，而是解决三个不同层面的问题：

- `FSD` 解决未知生成器和 few-shot 泛化问题
- `Stay-Positive` 解决判别依据不稳、过度依赖真实图像特征的问题
- `LVLM` 解决困难样本、冲突样本和边界样本的语义解释问题

因此，联合方案的真正目标是：

- 在统一训练框架下同时利用三类信息
- 让主干模型既保留泛化能力，又减少判别偏差
- 并能够对复杂样本提供更强的语义解释能力

## 3. 角色分工

### FSD

- 主干方法
- 负责主检测任务
- 负责未知生成器条件下的泛化能力
- 作为联合训练中唯一在线主干网络

### Stay-Positive

- 约束来源
- 负责提供 fake-oriented 判别偏置
- 负责减少模型对真实图像特征的错误依赖
- 在第一版中更适合作为 teacher 分数或辅助损失来源

### LVLM

- 语义辅助来源
- 负责给复杂样本生成结构化语义标签
- 负责训练后的案例解释和训练中的难样本辅助监督
- 在第一版中采用离线标签方式接入，不做端到端在线训练

## 4. 推荐联合层次

## 4.1 第一层：训练主干联合

第一版最先落地的联合层次是：

- 以 `FSD` 为主干
- 以 `Stay-Positive` 为辅助监督

这一步的目标是尽快拿到“不是只有基线复刻”的第一版联合结果。

## 4.2 第二层：语义辅助联合

在第一层稳定后，再接入：

- `LVLM` 语义辅助标签

这一步的目标是：

- 让主干特征具备一定局部异常敏感性
- 让论文中的解释性增强不只是后验文本分析

## 4.3 第三层：样本级难例联合

在前两层都跑通后，再进一步做：

- 冲突样本重加权
- 高风险异常样本强化训练

这样能够把已有的 `ADM` 冲突统计资产真正转化为训练增量。

## 5. 当前推荐方案

当前最推荐、最现实、最可写进论文的方法不是：

- 三套大模型端到端完全并行联合反传

而是：

- 主干：`FSD`
- 判别约束：`Stay-Positive`
- 语义辅助：`LVLM`

即采用“主干 + 判别约束 + 语义辅助”的分层联合设计。

## 6. 推荐损失设计

统一损失建议写成：

```text
L_total = L_fsd + λ1 * L_sp + λ2 * L_lvlm + λ3 * L_hard
```

其中：

- `L_fsd`：主任务分类 / prototypical loss
- `L_sp`：Stay-Positive 分数蒸馏或排序一致性损失
- `L_lvlm`：LVLM 语义辅助多标签损失
- `L_hard`：难样本重加权损失

## 7. 为什么不直接做三大模型完全联合反传

当前阶段不推荐这样做，主要因为：

1. 工程复杂度过高
2. 显存和资源成本过高
3. 调参空间过大，风险太高
4. 论文短期内更需要先拿到第一版联合结果，而不是构建最复杂但跑不完的系统

因此，最合理的路线是：

- 先做能跑通、能对比、能写进论文的第一版
- 再逐步增强

## 8. 当前最合适的首个训练落点

第一版建议优先聚焦：

- `ADM`
- `Midjourney`
- `SD`
- `real`

原因：

- `ADM` 是当前最关键困难类
- `Midjourney` 与 `SD` 是已有稳定正式对照
- 四类组合已经足以支撑第一版联合实验表
