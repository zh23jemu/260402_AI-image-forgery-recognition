# FSD 微调方案

## 1. 当前判断

当前已经完成两轮与 `ADM` 相关的训练探索：

- 一轮从零初始化训练
- 一轮基于官方 `converted checkpoint` 的小规模微调

目前判断已经比较明确：

- 不再建议继续随机初始化长训
- 继续沿着“基于官方 `converted checkpoint` 的保守微调”推进更稳妥

## 2. 为什么改成微调

当前切换到微调主线，主要原因有三点：

- 官方 checkpoint 已经明显强于当前从零初始化结果
- 从零初始化训练目前更像“训练链路验证”，而不是“性能提升方案”
- 论文中更容易把微调结果写成“在已有强基线基础上的小规模适配探索”

## 3. 首轮微调结果

当前已完成首轮微调结果如下：

| 设置 | Midjourney | Stable Diffusion | GLIDE | VQDM | ADM |
| --- | --- | --- | --- | --- | --- |
| 从零初始化训练 | 71.26% / 73.80% | 75.73% / 78.73% | 98.96% / 99.60% | 84.14% / 89.77% | 63.82% / 64.99% |
| 基于官方 checkpoint 微调 | 81.77% / 86.05% | 95.44% / 97.95% | 99.97% / 99.99% | 98.03% / 99.20% | 75.28% / 78.54% |

对 `ADM` 的关键比较为：

- 从零初始化：`63.82% / 64.99%`
- 首轮微调：`75.28% / 78.54%`
- 官方基线：`75.41% / 79.34%`

这说明首轮微调已经显著优于随机初始化训练，但还没有稳定超过官方基线。

## 4. 第二轮更稳推荐设置

第二轮更稳配置建议如下：

- `EXCLUDE_CLASS=ADM`
- `INIT_CKPT_PATH=../checkpoints/fsd/resnet50_exclude_adm_step[200000]_converted.pth`
- `LR=5e-6`
- `TOTAL_TRAINING_STEPS=5000`
- `SAVE_INTERVAL=5000`
- `EVAL_INTERVAL=5000`
- `BATCH_SIZE=16`
- `GPU_NUM=1`
- `OUTPUT_PATH=./output/finetune_adm_stage2`

这样设置的原因是：

- 当前首轮微调已经接近官方基线，不适合做激进改动
- 更小学习率更适合在强基线上做轻量修正
- 更短步数可以降低过拟合和指标回摆风险
- 保存和评估间隔与训练步数对齐，保证第二轮结束时一定产出 checkpoint 和最终评估结果
- 这一轮更适合验证“能否把 `ADM` 再轻推一步”

## 5. 第二轮观察目标

第二轮完成后，优先看：

1. `ADM` 是否高于当前从零初始化训练结果
2. `ADM` 是否稳定接近或超过当前官方 checkpoint 基线
3. `Midjourney`、`SD`、`GLIDE`、`VQDM` 是否维持首轮微调的整体提升
4. 是否值得再做第三轮极小幅调参

## 6. 当前口径

论文和实验记录里，对当前这条训练线更合适的表述是：

- “在已完成训练链路打通后，进一步尝试基于官方 checkpoint 进行小规模微调”
- “首轮微调已经明显优于从零初始化训练，说明强基线初始化是必要的”
- “第二轮配置用于验证是否能够在更保守设置下稳定逼近或略超官方基线”

不建议表述为：

- “已经完成最终改进模型训练”
- “已经稳定优于官方基线”
