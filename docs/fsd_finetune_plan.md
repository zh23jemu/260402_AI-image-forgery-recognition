# FSD 微调方案

## 1. 当前判断

当前已经完成三轮与 `ADM` 相关的训练探索：

- 一轮从零初始化训练
- 一轮基于官方 `converted checkpoint` 的小规模微调
- 一轮更保守的第二轮微调验证

目前判断已经比较明确：

- 不再建议继续随机初始化长训
- 继续沿着“基于官方 `converted checkpoint` 的保守微调”推进更稳妥

## 2. 为什么改成微调

当前切换到微调主线，主要原因有三点：

- 官方 checkpoint 已经明显强于当前从零初始化结果
- 从零初始化训练目前更像“训练链路验证”，而不是“性能提升方案”
- 论文中更容易把微调结果写成“在已有强基线基础上的小规模适配探索”

## 3. 首轮微调结果

当前已完成首轮微调结果如下：

| 设置 | Midjourney | Stable Diffusion | GLIDE | VQDM | ADM |
| --- | --- | --- | --- | --- | --- |
| 从零初始化训练 | 71.26% / 73.80% | 75.73% / 78.73% | 98.96% / 99.60% | 84.14% / 89.77% | 63.82% / 64.99% |
| 首轮基于官方 checkpoint 微调 | 81.77% / 86.05% | 95.44% / 97.95% | 99.97% / 99.99% | 98.03% / 99.20% | 75.28% / 78.54% |
| 第二轮保守微调 | 85.49% / 90.08% | 96.03% / 98.28% | 99.96% / 99.99% | 97.82% / 99.23% | 74.13% / 76.89% |

对 `ADM` 的关键比较为：

- 从零初始化：`63.82% / 64.99%`
- 首轮微调：`75.28% / 78.54%`
- 第二轮保守微调：`74.13% / 76.89%`
- 官方基线：`75.41% / 79.34%`

这说明首轮微调已经显著优于随机初始化训练，但还没有稳定超过官方基线；而第二轮保守微调没有进一步提升，反而略有回落。

## 4. 第二轮结果判断

第二轮配置为：

- `EXCLUDE_CLASS=ADM`
- `INIT_CKPT_PATH=../checkpoints/fsd/resnet50_exclude_adm_step[200000]_converted.pth`
- `LR=5e-6`
- `TOTAL_TRAINING_STEPS=5000`
- `SAVE_INTERVAL=5000`
- `EVAL_INTERVAL=5000`
- `BATCH_SIZE=16`
- `GPU_NUM=1`
- `OUTPUT_PATH=./output/finetune_adm_stage2`

第二轮结果说明：

- 该配置成功产出了完整 checkpoint 和最终评估结果
- 但在 `ADM` 上仅得到 `74.13% / 76.89%`，未能超过首轮微调
- 因此，“更小学习率 + 更短步数”在当前问题上并没有带来更稳的 `ADM` 提升
- 现阶段更合理的结论是：首轮微调已是当前最优训练探索结果，第二轮结果主要用于说明微调收益并不单调

## 5. 当前结论与后续建议

当前更建议优先做：

1. 把首轮微调保留为当前最优训练探索结果写入论文
2. 把第二轮结果写成“保守调参未进一步改善”的负结果
3. 优先转向结果分析、误判样本检查和 `LVLM` 案例补充
4. 只有在确实还需要一个最小训练尝试时，再考虑第三轮极小改动

## 6. 当前口径

论文和实验记录里，对当前这条训练线更合适的表述是：

- “在已完成训练链路打通后，进一步尝试基于官方 checkpoint 进行小规模微调”
- “首轮微调已经明显优于从零初始化训练，说明强基线初始化是必要的”
- “第二轮保守微调未进一步提升 `ADM`，说明当前训练收益对配置变化较为敏感”

不建议表述为：

- “已经完成最终改进模型训练”
- “已经稳定优于官方基线”
