# FSD 微调方案

## 1. 当前判断

当前已经完成一轮 `FSD / ADM` 从零初始化训练，但结果明显弱于官方 `converted checkpoint`。

因此，后续不再建议继续从随机初始化直接长训，而应切换到：

- 基于官方 `converted checkpoint`
- 做小规模、低学习率微调

## 2. 为什么改成微调

当前切换到微调主线，主要原因有三点：

- 官方 checkpoint 已经明显强于当前从零初始化结果
- 从零初始化训练目前更像“训练链路验证”，而不是“性能提升方案”
- 论文中更容易把微调结果写成“在已有强基线基础上的小规模适配探索”

## 3. 当前推荐设置

首轮微调建议如下：

- `EXCLUDE_CLASS=ADM`
- `INIT_CKPT_PATH=../checkpoints/fsd/resnet50_exclude_adm_step[200000]_converted.pth`
- `LR=1e-5`
- `TOTAL_TRAINING_STEPS=10000`
- `BATCH_SIZE=16`
- `GPU_NUM=1`

## 4. 观察目标

本轮微调完成后，优先看：

1. `ADM` 是否高于当前从零初始化训练结果
2. `ADM` 是否接近或超过当前官方 checkpoint 基线
3. 其他类别是否出现明显退化

## 5. 当前口径

论文和实验记录里，对这一轮更合适的表述是：

- “在已完成训练链路打通后，进一步尝试基于官方 checkpoint 进行小规模微调”
- “该实验用于验证基于强基线初始化的轻量适配是否优于从零初始化训练”

不建议表述为：

- “已经完成最终改进模型训练”
- “已经稳定优于官方基线”
