# 联合训练消融实验计划

## 1. 文档目标

本计划用于定义三模型联合训练的第一版结果表和最小消融组合，确保后续代码实现不会跑成“只有一个版本、无法证明改动有效”的状态。

## 2. 消融的基本原则

第一版消融不追求全面，只追求回答最核心的三个问题：

1. 加入 `Stay-Positive` 约束是否有帮助
2. 加入 `LVLM` 语义辅助是否继续有帮助
3. 难样本重加权是否对 `ADM` 这类困难类别有效

## 3. 推荐实验分组

第一版建议至少做以下 4 组：

### A 组：`FSD official`

作用：

- 作为历史正式基线
- 用于和后续所有联合结果比较

### B 组：`FSD finetune`

作用：

- 作为当前自训练基线
- 用于区分“普通微调提升”和“联合训练提升”

### C 组：`FSD + SP`

作用：

- 验证 `Stay-Positive` 约束是否能改善主干判断
- 这是第一版最关键的新增实验

### D 组：`FSD + SP + LVLM`

作用：

- 验证引入语义辅助监督后是否能继续改善
- 这是“三模型联合训练”真正完整的版本

## 4. 如果资源非常紧张

如果当前资源不足，最小也要保证：

1. `FSD official`
2. `FSD finetune`
3. `FSD + SP`

也就是说，第一阶段至少要先把 `SP` 约束跑出来。

## 5. 第二层消融

如果第一版顺利，建议补两个更细消融：

### E 组：`FSD + SP + hard_weight`

作用：

- 验证困难样本加权是否能单独带来收益

### F 组：`FSD + LVLM`

作用：

- 验证没有 `SP` 时，单独加入 LVLM 语义辅助是否有效

这个组可以不作为第一优先级，但如果资源允许，论文会更完整。

## 6. 指标建议

第一版建议至少汇报：

- `Accuracy`
- `AP`
- `ADM` 子集结果
- `Midjourney` 子集结果
- `SD` 子集结果

如果时间允许，再补：

- 冲突样本准确率
- 高权重难样本准确率

## 7. 推荐核心结果表

建议论文里至少出现一张这样的表：

| 方法 | Midjourney | SD | ADM | Avg |
| --- | --- | --- | --- | --- |
| FSD official | - | - | - | - |
| FSD finetune | - | - | - | - |
| FSD + SP | - | - | - | - |
| FSD + SP + LVLM | - | - | - | - |

其中：

- `ADM` 是重点观察列
- `Avg` 用于整体呈现

## 8. 推荐辅助结果表

建议再加一张聚焦困难样本的表：

| 方法 | ADM conflict subset | ADM hard subset | 备注 |
| --- | --- | --- | --- |
| FSD official | - | - | 历史基线 |
| FSD + SP | - | - | 判别约束 |
| FSD + SP + LVLM | - | - | 完整联合 |

这样更容易说明：

- 联合方法不是只在简单样本上抬分
- 而是在高价值困难样本上发挥作用

## 9. 实验优先级

建议优先级如下：

1. `FSD official`
2. `FSD finetune`
3. `FSD + SP`
4. `FSD + SP + LVLM`
5. `FSD + SP + hard_weight`
6. `FSD + LVLM`

## 10. 当前结论

对于当前项目而言，最重要的不是一次跑很多组，而是先保证：

- 至少有一组真实联合训练结果
- 至少有一张能够证明“不是只有基线复现”的核心对比表

所以第一步最优先的实验目标仍然是：

- `FSD + SP`

只要这一组跑出来，论文和客户口径就都会明显更稳。
