# 下一步建议

## 1. 当前判断

当前项目已经不能再把“基线复现、结果比较、案例分析”作为最终主线。

原因很明确：

- `FSD`
- `Stay-Positive`
- `LVLM`

这三条路线本身都来自已有论文。

如果最终论文主体仍停留在：

- 跑通已有代码
- 汇总已有结果
- 做少量案例观察

那么创新性会明显不足，也无法满足客户已经明确提出的“三模型联合训练”要求。

因此，当前阶段的默认主线必须切换为：

- 以 `FSD` 为主干
- 引入 `Stay-Positive` 的 fake-oriented 判别约束
- 接入 `LVLM` 的语义辅助标签
- 形成真正可写进论文的联合训练框架

## 2. 当前默认主线

当前默认优先工作对象已经更新为：

- 默认优先工作对象：`三模型联合训练`
- 默认优先主干：`FSD`
- 默认优先增强：`Stay-Positive`
- 默认优先辅助：`LVLM`
- 默认优先困难类别：`ADM`

这样安排的原因是：

- `FSD` 当前已经有可运行训练脚本
- `Stay-Positive` 已经积累了分数、阈值和冲突样本资产
- `LVLM` 已经积累了案例观察与语义分析入口
- `ADM` 是当前最能暴露方法差异、也最适合作为联合训练首个落点的困难类别

## 3. 现在最应该做的事

当前最重要的工作不是继续补传统基线文书，而是尽快把“联合训练”从概念变成最小可运行结果。

建议按以下顺序推进：

1. 先定义统一训练元数据格式
2. 先导出 `Stay-Positive` 对训练子集的离线分数
3. 再整理 `LVLM` 的结构化语义标签
4. 在 `fsd` 代码侧新建联合训练入口
5. 先跑 `FSD + Stay-Positive` 最小闭环
6. 再加 `LVLM` 辅助头

## 4. 当前不再推荐的路线

以下路线不再建议作为默认主线：

- 继续只做 `ADM` 案例补充
- 继续只做 `FSD` 单独微调
- 继续只改论文文本而不拿联合结果
- 继续停留在“基线比较 + 后续设想”

这些内容仍然可以保留，但它们现在只能作为：

- 联合方法提出之前的动机与基础
- 联合训练的前置证据

而不能再作为论文最终贡献本身。

## 5. 联合训练的最小闭环

当前最稳妥、最值得立刻动手的是：

- `FSD + Stay-Positive`

原因：

- `FSD` 训练主干已经存在
- `Stay-Positive` 已有现成预测分数可导出
- 无需先解决大模型在线训练问题
- 更容易在短时间内拿到第一版新增结果

第一版目标不是一步做到最复杂，而是先完成：

- 统一数据
- 统一训练入口
- 统一损失函数
- 至少一组能和 `FSD official` / `FSD finetune` 对比的结果

## 6. 第一阶段之后再做什么

在 `FSD + Stay-Positive` 跑通之后，第二阶段再接入：

- `LVLM` 语义辅助标签

其作用不是替代主分类器，而是：

- 给困难样本提供局部异常标签
- 给辅助头提供训练监督
- 给论文提供“为什么联合后更好”的解释证据

## 7. 当前建议

现在最稳妥的推进方式是：

1. 文档线以联合训练为中心口径全部同步
2. 代码线立即进入 `FSD + SP` 的联合训练最小闭环
3. 结果线优先盯住 `ADM + Midjourney + SD + real`
4. 等第一版联合结果出来后，再决定是否补更复杂的 `LVLM` 训练分支

## 8. 对外统一口径

当前对客户、导师或答辩场景，建议统一表述为：

“目前基线复现、结果比较与案例分析已经完成，这些工作将作为联合训练的前置基础。下一阶段的核心任务不是继续单独补基线，而是以 FSD 为主干，引入 Stay-Positive 的判别约束，并结合 LVLM 的语义辅助标签，完成第一版三模型联合训练验证。”
