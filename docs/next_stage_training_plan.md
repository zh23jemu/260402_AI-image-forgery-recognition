# 下一步更稳微调方案

## 1. 当前结论

当前文书、基线结果和方法口径已经基本收束完成，且 `ADM` 首轮微调已经跑完。

当前下一步不再是“做第一轮微调”，而是进入更保守的第二轮验证：

- 方法：`FSD`
- 目标类别：`ADM`
- 训练性质：基于官方 checkpoint 的第二轮稳健微调

## 2. 为什么继续做 FSD / ADM

选择这条训练主线，主要出于以下考虑：

- `FSD` 是当前论文中的主干路线
- 当前六类结果中，`ADM` 属于相对较弱类别之一
- `Stay-Positive` 在 `ADM` 上暴露出明显阈值问题，更适合作为对照和分析来源
- 首轮微调已经证明这条路线优于随机初始化训练，值得再做一轮更稳验证

## 3. 当前阶段判断

当前已知关键结果如下：

1. 从零初始化训练在 `ADM` 上为 `63.82% / 64.99%`
2. 首轮微调在 `ADM` 上为 `75.28% / 78.54%`
3. 官方基线在 `ADM` 上为 `75.41% / 79.34%`
4. 因此，首轮微调已经明显优于随机初始化训练，但尚未稳定超过官方基线

## 4. 第二轮推荐训练配置

当前建议的第二轮训练配置如下：

- `EXCLUDE_CLASS=ADM`
- `INIT_CKPT_PATH=../checkpoints/fsd/resnet50_exclude_adm_step[200000]_converted.pth`
- `GPU_NUM=1`
- `NUM_WORKERS=8`
- `BATCH_SIZE=16`
- `LR=5e-6`
- `TOTAL_TRAINING_STEPS=5000`
- `USE_FP16=True`
- `OUTPUT_PATH=./output/finetune_adm_stage2`

这轮的目标不是做大改，而是用更小步长做一次更稳的轻推。

## 5. 训练输出建议

建议第二轮训练输出目录使用清晰命名，例如：

```text
fsd/output/finetune_adm_stage2
```

日志文件建议单独保留，便于后续写论文时引用。

## 6. 训练后的第一批检查项

第二轮训练结束后，优先检查以下内容：

1. 训练是否完整跑完
2. 输出目录中是否生成 checkpoint
3. 用新 checkpoint 在 `ADM` 上重新评估
4. 与首轮微调结果做对比
5. 与当前 `resnet50_exclude_adm_step[200000]_converted.pth` 做对比
6. 判断是否已经足够支撑论文中的“轻量适配探索”表述

## 7. 训练后文书补位

第二轮训练完成后，第一时间需要同步更新：

- `docs/experiment_results_summary.md`
- `docs/full_paper_draft.md`
- `weekly_report/` 中对应周报

补的重点不是堆细节，而是：

- 是否跑通
- 指标有没有变化
- 变化是否支持继续推进

## 8. 当前建议

现在最稳妥的推进方式是：

1. 直接运行第二轮更稳配置
2. 重点盯住 `ADM` 与首轮微调的差异
3. 若结果持平或略升，就优先用于论文收束
4. 再决定要不要做第三轮极小改动
