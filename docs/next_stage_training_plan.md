# 下一步最小训练方案

## 1. 当前结论

当前文书、基线结果和方法口径已经基本收束完成，下一步可以正式进入最小改进训练。

默认首个训练对象建议为：

- 方法：`FSD`
- 目标类别：`ADM`
- 训练性质：最小改进训练

## 2. 为什么先训 FSD / ADM

选择这条训练主线，主要出于以下考虑：

- `FSD` 是当前论文中的主干路线
- 当前六类结果中，`ADM` 属于相对较弱类别之一
- `Stay-Positive` 在 `ADM` 上暴露出明显阈值问题，更适合作为对照和分析来源
- 先做 `FSD` 的最小训练，更容易得到一条清晰、可写进论文的“训练增量结果”

## 3. 当前训练目标

首轮训练不追求一步到位，而是先完成以下目标：

1. 让训练链路稳定跑通
2. 获得一条自训练模型结果
3. 与当前 `ADM` 官方 converted checkpoint 结果做对比
4. 判断训练是否值得继续扩展

## 4. 推荐训练配置

当前建议的首轮训练配置如下：

- `EXCLUDE_CLASS=ADM`
- `GPU_NUM=1`
- `NUM_WORKERS=8`
- `BATCH_SIZE=16`
- `LR=1e-4`
- `TOTAL_TRAINING_STEPS=50000`
- `USE_FP16=True`

如果想先做更低风险试跑，可先把：

- `TOTAL_TRAINING_STEPS` 调成 `5000` 或 `10000`

用于检查训练是否稳定。

## 5. 训练输出建议

建议首轮训练输出目录使用清晰命名，例如：

```text
fsd/output/train_adm_stage1
```

日志文件建议单独保留，便于后续写论文时引用。

## 6. 训练后的第一批检查项

首轮训练结束后，优先检查以下内容：

1. 训练是否完整跑完
2. 输出目录中是否生成 checkpoint
3. 用新 checkpoint 在 `ADM` 上重新评估
4. 与当前 `resnet50_exclude_adm_step[200000]_converted.pth` 做对比
5. 判断指标变化是否值得继续扩展到其他类别

## 7. 训练后文书补位

训练完成后，第一时间需要同步更新：

- `docs/experiment_results_summary.md`
- `docs/full_paper_draft.md`
- `weekly_report/` 中对应周报

补的重点不是堆细节，而是：

- 是否跑通
- 指标有没有变化
- 变化是否支持继续推进

## 8. 当前建议

现在最稳妥的推进方式是：

1. 先按训练前检查清单确认环境
2. 跑一轮 `FSD / ADM` 最小训练
3. 评估结果
4. 再决定要不要做第二轮扩展或更复杂方案
