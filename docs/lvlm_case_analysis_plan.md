# LVLM 轻量案例分析方案

## 1. 文档目的

这份文档用于把 `LVLM` 从“论文中已经纳入的研究线”推进到“可以实际执行的小规模验证任务”。

当前目标不是做大规模定量评测，而是完成一组足以支撑论文写作和老师汇报的定性案例分析。

## 2. 分析目标

`LVLM` 轻量案例分析主要服务于以下三个目标：

- 解释传统判别模型为什么会在某些样本上出错
- 展示高层语义分析对复杂伪造图像识别的补充价值
- 为论文中的“方法边界”和“后续工作”提供更具体的案例支撑

## 3. 样本来源优先级

后续选样时，优先顺序建议如下：

### 3.1 第一优先级：FSD 误判样本

优先挑选：

- `FSD` 在 `ADM`、`VQDM` 等低表现类别上的误判样本
- 预测与真实标签明显不一致的样本
- 看起来“肉眼不容易一眼判断”的样本

这类样本最适合用于说明：

- 为什么跨生成器泛化仍然困难
- 为什么仅依赖现有判别分数还不够

### 3.2 第二优先级：模型冲突样本

优先挑选：

- `FSD` 判断为假，但 `Stay-Positive` 判断为真
- `FSD` 判断为真，但 `Stay-Positive` 判断为假
- 两个模型分数都接近边界、结论不稳定的样本

这类样本最适合用于说明：

- 不同方法实际关注的证据并不相同
- `LVLM` 可以作为辅助解释接口

### 3.3 第三优先级：边界样本

优先挑选：

- 分数接近阈值的样本
- 真假视觉上都比较接近自然图像的样本
- 局部细节疑似异常、但传统模型结论不够稳定的样本

这类样本最适合用于说明：

- 高质量伪造图像识别中的困难点
- `LVLM` 在复杂场景下的补充意义

## 4. 建议样本规模

当前最稳妥的样本规模建议为：

- 总样本数：`6` 到 `12` 张
- 每类样本：`2` 到 `4` 张

不建议一开始超过 `12` 张，因为：

- 后续人工整理成本会明显上升
- 论文现阶段更需要“代表性案例”，而不是“案例堆积”

## 5. 输出字段设计

每个案例建议统一整理以下字段：

- `sample_id`
- `image_path`
- `generator_type`
- `ground_truth`
- `fsd_prediction`
- `fsd_score`
- `stay_positive_prediction`
- `stay_positive_score`
- `case_type`
- `lvlm_prompt`
- `lvlm_response_summary`
- `human_takeaway`

其中：

- `case_type` 建议取值：
  - `fsd_misclassified`
  - `model_conflict`
  - `borderline_case`
- `human_takeaway` 用于最后写进论文

## 6. LVLM 输出关注点

让 `LVLM` 分析时，重点希望它覆盖以下维度：

- 图像整体语义是否自然
- 人物、物体、背景之间的关系是否合理
- 局部结构是否存在违和，如手部、边缘、纹理、反射、阴影
- 是否存在不符合常识的细节组合
- 如果必须给出判断，更倾向真实还是伪造，以及理由是什么

## 7. 论文中的落点

后续若完成 `LVLM` 轻量案例分析，最适合落在论文中的位置有两个：

### 7.1 第五章讨论部分

可作为：

- “误判样本分析”
- “复杂样本的语义解释补充”
- “为什么仅依赖分数仍然不够”

### 7.2 第六章后续工作部分

可作为：

- “LVLM 已具备初步验证入口”
- “后续可围绕边界样本开展系统性案例研究”

## 8. 最小执行步骤

实际执行时，建议按下面顺序推进：

1. 从现有日志或结果文件里筛出候选误判样本
2. 手动挑出 `6` 到 `12` 张代表性图片
3. 为每张图记录 `FSD` 与 `Stay-Positive` 的结论
4. 用统一提示词交给 `LVLM` 分析
5. 将输出整理成摘要，而不是直接贴原始长回答
6. 从中提炼 2 到 3 条可写入论文的共性观察

## 9. 当前建议结论

如果时间有限，`LVLM` 最值得做的不是大规模实验，而是：

- 先做一组高质量的小样本案例分析
- 重点服务论文讨论、结论和后续计划

这样既满足客户“必须纳入 `LVLM`”的要求，也不会把当前项目节奏拉乱。
