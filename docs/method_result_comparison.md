# FSD、Stay-Positive 与 LVLM 方法与结果对比

## 1. 方法定位对比

| 方法 | 当前使用模型 | 当前测试/使用方式 | 主要特点 | 当前阶段定位 |
| --- | --- | --- | --- | --- |
| FSD | 官方预训练 `ResNet-50` checkpoint | 基于 `GenImage` 目录按类评估；补充少量微调探索 | 强调少样本泛化与未知生成器检测 | 作为泛化检测主干基线 |
| Stay-Positive | `Corvi+` / `Rajan-Ours+` 预训练模型 | 先生成 CSV，再做 real/fake 打分评估 | 强调减少对真实图像特征的错误依赖 | 作为鲁棒判别主基线 |
| LVLM | 当前未固定单一模型 | 基于误判样本、冲突样本与边界样本开展定性分析 | 强调高层语义理解与解释性补充 | 作为复杂样本辅助分析路线 |

## 3. 当前结果的直接观察

- `FSD` 在不同生成器上的性能差异明显，其中 `GLIDE` 最好，`ADM` 与 `VQDM` 最弱。
- `Stay-Positive` 在当前 `Midjourney` 与 `SD` 的固定 real/fake 测试中显著优于 `FSD`。
- `Stay-Positive / ADM` 扩展观察提示其可能存在阈值或校准问题。
- `FSD / ADM` 首轮微调已经显著优于随机初始化训练，但第二轮保守微调未进一步提升。
- `LVLM` 当前不提供统一数值结果，但已经具备围绕 `ADM` 误判样本开展案例分析的执行入口。

## 4. 当前阶段应如何解释这些差异

- `FSD` 与 `Stay-Positive` 的方法目标并不完全相同，不能仅凭单组指标简单下结论说某一方法“全面更优”。
- 当前 real/fake 对照测试更接近 `Stay-Positive` 的直接应用场景，因此其结果更高是可以理解的。
- `FSD` 的研究价值更多体现在跨生成器泛化、少样本适应和后续方法融合空间上。
- 训练探索结果说明，强基线初始化是必要的，但后续微调收益对配置较为敏感。
- `LVLM` 当前最合适的角色是为误判样本、冲突样本和边界样本提供语义解释补充，而不是参与机械化数值排名。

## 5. 当前建议使用的结论

当前实验表明，`Stay-Positive` 在固定 real/fake 对照测试中表现出更强的直接判别能力；`FSD` 虽然在当前闭集指标上不占优势，但更适合作为面向未知生成器泛化与后续方法融合的主干路线；围绕 `ADM` 的训练探索进一步说明，基于官方 checkpoint 的首轮微调已经能够显著优于随机初始化训练，但第二轮更保守的微调设置未进一步改善结果。因此，当前最稳妥的论文表述应是：项目已经形成了较完整的多路线基线体系，训练探索已提供正反两类证据，而下一步工作更适合转向误判分析与 `LVLM` 小样本案例补充，而不是继续无边界追加微调实验。
