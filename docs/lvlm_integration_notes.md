# LVLM 纳入论文的集成说明

## 1. 当前结论

客户已经明确，`LVLM` 需要纳入本次论文。

因此，当前论文的整体方法框架应统一表述为：

- `FSD`：面向未知生成器泛化的主干基线
- `Stay-Positive`：面向鲁棒判别与伪相关抑制的增强基线
- `LVLM`：面向复杂场景语义分析、边界样本解释和辅助决策的第三条研究线

这不是可选扩展，而是本次论文的正式组成部分。

## 2. 当前做到哪里

截至目前，`LVLM` 的推进状态应统一表述为：

- 已完成论文结构纳入
- 已完成方法定位说明
- 已完成与 `FSD`、`Stay-Positive` 的关系梳理
- 已明确当前阶段不夸大为“已完成完整定量实验”

也就是说，当前 `LVLM` 已经被正式写入论文框架，但仍处于“方法分析 + 场景设计 + 轻量验证准备”的阶段。

## 3. 为什么要纳入 LVLM

将 `LVLM` 纳入本次论文，主要是出于以下考虑：

- 仅依赖传统真假分类分数，难以完整覆盖复杂场景下的伪造图像识别问题
- 当图像缺少明显低层伪迹时，需要更高层的语义一致性分析作为补充
- 论文若只写 `FSD` 和 `Stay-Positive`，在当前客户要求下会显得方法版图不完整
- `LVLM` 适合承担“解释性分析”和“边界样本辅助判断”的角色，能够提升论文整体结构完整性

## 4. 当前口径边界

后续在论文、周报、汇报或答辩中，关于 `LVLM` 的表述建议统一为：

- `LVLM` 已纳入本次论文整体研究框架
- 当前阶段重点是明确其研究定位、适用场景和可行性方案
- 现阶段已完成的是结构整合与方案设计，不是大规模正式数值实验
- 若时间允许，将补充轻量级案例分析或示例验证

不建议表述为：

- `LVLM` 已经完成系统性实验并与其他方法完成同口径比较
- `LVLM` 已经成为当前论文的主要数值结果来源

## 5. 后续工作顺序

当前推荐的 `LVLM` 后续工作顺序如下：

1. 先把论文主稿中的 `LVLM` 相关正文补完整
2. 再整理 `LVLM` 在本项目中的任务边界和应用场景
3. 然后设计轻量级案例分析入口
4. 最后视时间决定是否补 1 组小样本示例验证

## 6. 最稳妥的轻量验证方式

如果后续要做 `LVLM`，最稳妥的不是一上来做大规模评测，而是做小规模、可解释的案例验证，例如：

- 从 `FSD` 或 `Stay-Positive` 的误判样本中挑选若干代表性图片
- 从边界分数样本中挑选若干“真假难分”的案例
- 设计统一提示词，让 `LVLM` 输出语义异常、结构异常和解释性判断
- 将这些输出整理成论文中的定性分析材料

这样做的好处是：

- 风险低
- 写作收益高
- 更符合当前论文阶段的时间约束
- 能自然体现 `LVLM` 的研究作用
