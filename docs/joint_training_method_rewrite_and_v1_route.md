# 论文方法章节改写框架与第一版落地训练路线

## 1. 方法章节必须怎么改

如果客户明确要求“三模型联合训练”，那么论文第三章和第四章的中心叙事必须从：

- 基线方法介绍
- 结果比较
- 后续设想

改成：

- 联合框架提出
- 联合训练目标设计
- 联合数据构造
- 联合实验实现

也就是说，现有初稿中的：

- `FSD`、`Stay-Positive`、`LVLM` 的单独介绍
- 基线比较结果

仍然可以保留，但定位要改为：

- 联合方法提出之前的基础分析与动机来源

而不是全文最终贡献本身。

## 2. 第三章建议改写结构

当前第三章题目是：

- `基线方法与研究方案`

建议改写为：

- `三模型联合训练方法设计`

### 3.1 联合训练问题定义

写清楚：

- 本文不再把任务仅定义为单一二分类
- 而是把任务定义为“主检测 + 判别约束 + 语义辅助”的联合优化问题

本节要回答的问题：

- 为什么仅靠单模型复现不够
- 为什么要把三条路线联合起来
- 三条路线分别解决什么短板

### 3.2 联合框架总体结构

本节给出方法总览图，建议图中包括：

- `FSD backbone`
- `classification head`
- `Stay-Positive supervision branch`
- `LVLM semantic supervision branch`
- `hard sample reweighting`

本节文字应明确：

- `FSD` 是主干
- `Stay-Positive` 提供判别约束
- `LVLM` 提供语义辅助监督

### 3.3 FSD 主干检测模块

写法重点：

- 简要介绍当前主干沿用 `FSD` 的原因
- 说明其承担 few-shot / open-generator 主任务
- 说明本文不会完全重写 `FSD`，而是在其基础上做联合增强

### 3.4 Stay-Positive 判别约束模块

本节需要把 `Stay-Positive` 从“被比较对象”改写为“联合训练中的辅助监督来源”。

重点表述：

- `Stay-Positive` 不直接作为并行大主干端到端反传
- 而是为主干输出提供 fake-oriented 约束
- 可具体落在分数蒸馏、排序一致性或判别偏置正则化上

### 3.5 LVLM 语义辅助模块

重点写法：

- `LVLM` 的角色不是主分类器
- 而是复杂样本的语义标签生成器与训练辅助接口
- 其输出参与辅助头训练和难样本加权

### 3.6 联合损失函数设计

建议公式结构：

```text
L_total = L_fsd + λ1 * L_sp + λ2 * L_lvlm + λ3 * L_hard
```

本节必须逐项解释：

- 主任务损失
- SP 约束损失
- LVLM 辅助损失
- 难样本加权策略

### 3.7 训练与推理流程

训练流程：

1. 先离线生成 SP 分数与 LVLM 标签
2. 再用统一 metadata 驱动联合训练
3. 训练后输出主分类结果和辅助语义预测

推理流程：

- 在线阶段只保留 `FSD` 主干和轻量辅助头
- `LVLM` 可作为分析模式下的可选解释接口

### 3.8 方法优势分析

本节是论文里非常重要的一段，需要明确写出三点：

1. 相比单独 `FSD`，新方法对 fake-oriented 证据更敏感
2. 相比单独 `Stay-Positive`，新方法保留了未知生成器泛化主干
3. 相比只做后验分析的 `LVLM`，新方法把语义信息前移到了训练阶段

## 5. 第一版可落地训练路线

下面这条路线是当前最适合马上启动的版本。

## 5.1 第一阶段：先做 `FSD + SP`

这是最小闭环。

目标：

- 不先碰 LVLM 辅助头
- 先把 `Stay-Positive` 分数约束接进 `FSD`
- 拿到第一版真正的“联合训练结果”

具体工作：

1. 导出 `Stay-Positive` 对训练子集的分数
2. 给 `FSD` 训练样本对齐这些分数
3. 在 `fsd/train.py` 中增加 `L_sp`
4. 先跑 `real + ADM + SD + Midjourney`

预期收益：

- 能最先满足“不是只有基线复刻”
- 改动量最小
- 最容易写进论文

## 5.2 第二阶段：加 `LVLM` 语义辅助头

目标：

- 对 `ADM` 冲突样本加入语义辅助标签
- 增加一个 lightweight auxiliary head

具体工作：

1. 用现有案例池筛出高价值样本
2. 给这些样本补结构化语义标签
3. 在 `FSD` backbone 后增加辅助头
4. 加入 `L_lvlm`

预期收益：

- 真正形成“三模型联合训练”
- 论文方法层面更完整
- 可解释性更强

## 5.3 第三阶段：难样本重加权

目标：

- 利用已有冲突统计与误判样本池
- 对训练中的关键困难样本提高权重

具体工作：

1. 标出 `FSD` / `SP` 冲突样本
2. 标出 `LVLM` 高风险标签样本
3. 在 loss 中加入样本权重

预期收益：

- 更符合当前项目中 `ADM` 的真实问题形态
- 能把已有分析资产真正转化为训练增量
